## 3. 核心功能模块

### 3.1 AI 助手系统

**API 端点**: `POST /api/chat-openai`

| 功能 | 说明 |
|------|------|
| **自然语言交互** | 用户通过聊天记录健身/饮食数据 |
| **意图提取** | AI 返回结构化 JSON，包含 `intent` 和 `data` |
| **图片识别** | 支持上传食物图片，AI 识别并估算营养（多模态） |

**核心流程**:
```
用户输入 → 构建记忆上下文 → 调用 OpenAI API → 
解析 JSON 响应 → 保存到数据库 → 返回给前端
```

### 3.2 记忆系统 (`services/memoryService.ts`)

| 记忆类型 | 用途 | 存储位置 |
|---------|------|---------|
| **语义记忆** (Semantic) | 用户画像：目标、弱点、偏好、伤病史 | `user_semantic_memory` 表 |
| **场景记忆** (Episodic) | 最近活动记录摘要 | `activity_records` 表 |
| **工作记忆** (Working) | 当前会话上下文 | 内存中 |

**记忆更新机制**:
- AI 从对话中提取用户画像更新 → 调用 `updateSemanticMemory()`
- 每次对话自动保存到场景记忆 → `addChatMessage()`

### 3.3 意图系统

| 意图 | 描述 | 触发场景 |
|------|------|---------|
| `generate_workout_plan` | 创建训练计划 | "给我做个训练计划" |
| `update_workout_plan` | 调整现有计划 | "根据我的进度调整计划" |
| `log_strength_workout` | 记录力量训练 | "今天练了胸推" |
| `log_exercise` | 记录有氧运动 | "跑了5公里" |
| `log_food` | 记录饮食（文字） | "吃了牛肉面" |
| `log_food_multi` | 记录饮食（图片） | 上传食物照片 |
| `log_measurement` | 记录身体指标 | "体重72.5kg" |
| `chat` | 普通聊天 | 闲聊/提问 |

### 3.4 数据库 Schema

**核心表结构**:

```sql
-- 用户表
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 聊天会话表
CREATE TABLE chat_sessions (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  title TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  archived INTEGER DEFAULT 0,
  archived_at DATETIME,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 聊天消息表
CREATE TABLE chat_messages (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  image_data TEXT,
  intent TEXT,
  intent_data TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 活动记录表（统一存储各类记录）
CREATE TABLE activity_records (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  intent TEXT NOT NULL,
  entry_date TEXT,
  timestamp DATETIME NOT NULL,
  data_json TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 语义记忆表（用户画像）
CREATE TABLE user_semantic_memory (
  user_id TEXT PRIMARY KEY,
  memory_json TEXT NOT NULL,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 4. API 接口列表

### 4.1 聊天相关

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/system-prompt` | 获取 AI 系统提示词 |
| GET | `/api/chat/:sessionId` | 获取会话聊天记录 |
| POST | `/api/chat-openai` | 发送消息到 AI（核心接口） |
| GET | `/api/chat-sessions` | 获取会话列表（支持筛选） |
| POST | `/api/chat-sessions` | 创建新会话 |
| PATCH | `/api/chat-sessions/:sessionId` | 更新会话（重命名/归档） |
| DELETE | `/api/chat-sessions/:sessionId` | 删除会话 |

### 4.2 记录管理

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/save-record` | 保存健康记录到数据库 |
| GET | `/api/logs` | 获取所有活动记录 |
| PATCH | `/api/logs/:recordId` | 更新活动记录 |
| DELETE | `/api/logs/:recordId` | 删除活动记录 |

### 4.3 其他

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/upload-image` | 上传图片（Base64 转文件） |
| GET | `/api/semantic-memory` | 获取用户语义记忆（画像） |

## 5. 开发指南

### 5.1 环境要求

- Node.js 18+
- npm 9+
- SQLite3

### 5.2 环境变量配置

创建 `.env` 文件：

```env
# OpenAI 配置
OPENAI_API_KEY=your_openai_api_key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini

# Google Gemini 配置（可选）
GEMINI_API_KEY=your_gemini_api_key

# 默认 AI 提供商
DEFAULT_AI_PROVIDER=openai
```

### 5.3 常用命令

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 生产构建
npm run build

# 预览生产构建
npm run preview

# TypeScript 类型检查
npm run lint

# 清理构建目录
npm run clean
```

### 5.4 开发服务器启动流程

```
1. server.ts 加载环境变量
2. 初始化 Express 应用
3. 连接 SQLite 数据库（自动创建表）
4. 注册 API 路由
5. 根据环境配置 Vite 中间件（开发模式）
6. 启动 HTTP 服务器（端口 3000）
```

## 6. 代码质量评估

### 6.1 优势 ✅

| **TypeScript 类型安全** | 广泛使用接口和类型定义，减少运行时错误 |
